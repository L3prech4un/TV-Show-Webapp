API Reference¶

Database Query Functions¶

All database operations are handled through db/query.py. Key functions:

get_User(table, **filters)¶

Get a single user by filters.

Parameters:

  • table – Database table object

  • **filters –

    Keyword arguments for filtering

Returns:

User object or None

get_all(table)¶

Get all records from a table.

Parameters:

table – Database table object

Returns:

List of records

getFeed(userid)¶

Get feed posts for a user (posts from followed users + own posts).

Parameters:

userid – User ID

Returns:

List of post dictionaries

createPost(userid, mediaid, title, content)¶

Create a new post.

Parameters:

  • userid – User ID creating the post

  • mediaid – Media ID the post is about

  • title – Post title

  • content – Post content

Returns:

None

follow_user(user_id, follower_id)¶

Follow another user.

Parameters:

  • user_id – Current user ID

  • follower_id – User ID to follow

Returns:

Boolean indicating success

Database Models¶

User Model (db/schema/user.py):

  • UserID (Primary Key)

  • FName, LName, UName

  • Email, PWord (hashed)

  • Relationships: Post, Comment, TVMovie associations

Post Model (db/schema/post.py):

  • PostID (Primary Key)

  • MediaID (Foreign Key to TVMovie)

  • Title, Date, Content

  • Relationships: User, Comment, TVMovie
